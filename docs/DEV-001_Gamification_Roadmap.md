# DEV-001 游戏化功能开发路线图

> **定位**: 模拟娱乐商城增强体验 | **批次**: 5 批 | **预计总工时**: 每批 1-2 小时

---

## 分批计划

| 批次 | 功能 | 优先级 | 说明 |
|---|---|---|---|
| **第1批** | 优惠券系统 | P0 | 满减/折扣/百分比券，领券→下单抵扣，管理后台 CRUD |
| **第2批** | 积分商城 | P0 | 签到积分 + 购物积分 → 兑换虚拟商品，积分明细 |
| **第3批** | 秒杀/限时抢购 | P1 | 限时限量低价抢购，增加紧迫感 |
| **第4批** | 专题活动页 | P1 | "618大促"等运营专题，营造购物节氛围 |
| **第5批** | 消费统计 + 排行榜 | P2 | 消费榜、签到榜，增强社交趣味性 |

## 每批标准开发流程

1. **数据库** — 执行建表 + 初始化数据 SQL
2. **后端** — Entity → Mapper → Service → Controller
3. **前端** — API 模块 → 页面 → 路由 → 菜单
4. **编译** — `mvn compile` 验证后端无报错
5. **启动** — `mvn spring-boot:run` 验证启动成功
6. **功能测试** — 用浏览器/Postman 测试每个接口
7. **修复** — 发现 Bug 立即修复，重新测试直到通过

## 技术约束

- 项目定位：纯模拟商城，所有支付/物流均为模拟
- 后端：Spring Boot 3 + MyBatis-Plus
- 前端：Vue 3 + TypeScript + Element Plus
- 缓存：优先走 CacheProviderFactory（支持 Redis/DB 切换）
- 权限：Controller 类级 @PreAuthorize
